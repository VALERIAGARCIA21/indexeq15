¿Que es CSS?

CSS esta denomina lenguaje de hojas estilo cascada, fue desarrollado por WC3 en 1996 por una razón sencilla. Y es que el HTML no esta diseñado para tener etiquetas que ayuden a formatear la pagina.

¿Para que sirve el CSS?

El CSS utiliza una sintaxis simple basada en el inglés con un conjunto de reglas que la gobiernan. Dado a que el HTML no fue hecho con la intención de utilizar elementos de estilo, sino para el marcado de la página. Se creo para describir el contenido.

Se puede crear reglas para decirle al sitio web como desea mostrar la información y se puede guardar los comandos para los elementos de estilo separados de los que configuran el contenido.

El CSS se conoce como si fuera el propietario de sitio o experto en marketing con conocimiento web es conveniente que conozcas algunos aspectos fundamentales.

¿Cuál es la diferencia que sería más difícil de realizar en un sitio web si se optara por no usar un CSS?

Se dice que el HTML define la forma y estrucutra de una pagina web, mientras que el CSS es un lenguaje que funciona a traves de hojas de diseño, yaq ue decide la apariencia visual a traves de los estilos.

El papel del archivo de reset CSS

Todos los elementos HTML se muestran en pantalla utilizando los estilos por defecto que aplica cada navegador, donde cada navegador aplica sus propias reglas para mostrar los elementos en pantalla. Estas diferencias son las que permiten que en principio los navegadores muestren las páginas en formas distintas.

A raíz de estas diferencias, Eric Meyer introdujo en el mundo del diseño y programación web el concepto de “hoja de estilo reset” (Reset Stylesheet). Su idea era elaborar una plantilla base que garantizara que cada navegador, aplicaría las mismas reglas a los elementos de una página.

Breve Historia de las Tecnicas

Las hojas de estilos aparecieron poco después que el lenguaje de etiquetas SGML, alrededor del año 1970. Desde la creación de SGML, se observó la necesidad de definir un mecanismo que permitiera aplicar de forma consistente diferentes estilos a los documentos electrónicos. El organismo W3C (World Wide Web Consortium), encargado de crear todos los estándares relacionados con la web, propuso la creación de un lenguaje de hojas de estilos específico para el lenguaje HTML y se presentaron nueve propuestas. Las dos propuestas que se tuvieron en cuenta fueron la CHSS (Cascading HTML Style Sheets) y la SSP (Stream-based Style Sheet Proposal).

A finales de 1996, el W3C publicó la primera recomendación oficial, conocida como "CSS nivel 1". A principios de 1997, el W3C decide separar los trabajos del grupo de HTML en tres secciones: el grupo de trabajo de HTML, el grupo de trabajo de DOM y el grupo de trabajo de CSS. El 12 de Mayo de 1998, el grupo de trabajo de CSS publica su segunda recomendación oficial, conocida como "CSS nivel 2". Al mismo tiempo, la siguiente recomendación de CSS, conocida como "CSS nivel 3", continúa en desarrollo desde 1998 y hasta el momento sólo se han publicado borradores.

El elemento div: creacion de una estructura CSS de dos columnas con anchura fija

El div es un elemento de bloque que sirve como contenedor genérico, en el que podemos colocar varios elementos relacionados. Una página puede tener los div’s que sean necesarios, y es muy común que estos contengan alguna ID o pertenezcan a una clase, como medio para aplicarles un estilo y acomodarlos de forma conveniente. De igual forma son útiles para separar e identificar las distintas secciones de la página web, como la barra del menú, un índice, un bloque de texto, etc.

Algunas características esenciales del div es que, al ser un elemento de bloque, ocupa todo el ancho disponible dentro de su contenedor, o sea que se extiende hasta donde puede, y siempre comienza en una nueva línea, por ende al crear varios de estos se acomodan automáticamente uno abajo del otro. Sin embargo, claro que es posible modificar su altura y anchura, y cambiar sus posiciones dentro del contenedor mediante una hoja de estilos, como también veremos a continuación.

Columnas y Propiedad Float CSS

Esta propiedad permite que elementos “floten” en una posición específica de un contenedor, haciendo que los demás elementos cambien su posición respecto a éste. Un ejemplo podría ser aplicarle la propiedad float a una imagen para que cierto texto la rodee. La propiedad float solo acepta los valores “right”, “left”, “none” e “inherit”; asi que los elementos no pueden flotar en el centro.

Con tal propiedad también se pueden tener varios elementos flotantes en un mismo contenedor, de forma que se alinean unos a lado de otros y siguen en una nueva línea al llegar al final, esto aplica también para los elementos div (a pesar de ser elementos de bloque como se explicó anteriormente) y es una característica muy usada para barras de menú horizontales, galerías de imágenes e incluso columnas de texto.

En cuanto a este último ejemplo, se puede lograr mediante varios contenedores div con la propiedad float, en los cuales colocaríamos todo el texto que se quiere acomodar, y eso daría el efecto de columnas como el de una página de Word, por ejemplo. Igualmente, para la galería se colocarían varias imágenes flotantes, y para el menú un contenedor con varios elementos “button” con esta misma propiedad.

La propiedad clear

La propiedad CSS clear especifica si un elemento puede estar al lado de elementos flotantes que lo preceden o si debe ser movido (cleared) debajo de ellos. La propiedad clear aplica a ambos elementos flotantes y no flotantes.

Cuando es aplicado a bloques no flotantes, mueve el border edge (en-US) del elemento hacia abajo hasta que este debajo del margin edge (en-US) de todos los floats relevantes. Este movimiento (cuando acontece) causa que margin collapsing (en-US) no ocurra.

Cuando se aplica a elementos flotantes, mueve el margin edge (en-US) del elemento debajo del margin edge (en-US) de todos los floats relevantes. Esto afecta la posición de floats posteriores, ya que estos no pueden ser posicionados más arriba que los primeros.

Los floats que son relevantes para ser limpiados (cleared) son los primeros floats dentro del mismo contexto de formato de bloque.

Añadir estilos para el texto

Como HTML, CSS (Cascading Style Sheets) u Hojas de estilo en cascada en español, no es realmente un lenguaje de programación, tampoco es un lenguaje de marcado. Es un lenguaje de hojas de estilo, es decir, te permite aplicar estilos de manera selectiva a elementos en documentos HTML.

Hay tres formas de aplicar estilos CSS en un documento HTML: en línea, incrustado en la cabecera y mediante hojas de estilo externas.

  • CSS en linea

    • Los estilos en línea son declaraciones CSS que se integran en las etiquetas HTML mediante el atributo style. Este método tan solo afecta al elemento en el que se integra el código. El CSS en línea es complicado de entender y mantener ya que mezcla los estilos CSS con el código HTML.

  • CSS incrustado en la cabecera

    • Otra manera muy simple de añadir estilo con CSS es utilizando la etiqueta (style) en la cabecera head del fichero HTML del sitio. La desventaja de este método es que, a la hora de realizar cualquier cambio, se debe realizar en múltiples páginas diferentes y el código estará repetido. Su uso puede llegar a ser necesario en el caso de utilizar un gestor de contenido que no permita modificar el archivo CSS directamente.

  • CSS en Hojas de estilo externas

    • Mediante hojas de estilo externas se consigue separar el archivo de estilos del fichero HTML. El archivo de estilos cuenta con la extensión .css y se referencia desde HTML mediante el elemento . Este es el método más eficiente y más sencillo de mantener ya que el código CSS se encuentra separado del fichero HTML.
